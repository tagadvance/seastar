PLAN: Support UPDATE ... SET ... WHERE (UpdateHandler)

DEPENDS ON: a_create_table_persistence, b_insert, c_select_where (reuses WHERE matching + row model).
NOTE: In Cassandra UPDATE is an upsert; INSERT and UPDATE share UpdateStatement machinery.

READ FIRST
- Skill: .claude/skills/add-cql-statement/SKILL.md
- InsertHandler.java (from plan b) and SelectHandler WHERE logic (from plan c) -- reuse both.

INSPECT
  ./gradlew :lib:inspectRaw -Pquery="UPDATE foo.users SET first_name = ? WHERE user_id = ?" --console=plain -q 2>/dev/null
  ./gradlew :lib:inspectRaw -Pquery="UPDATE foo.users SET first_name = 'Ann' WHERE user_id = ? IF EXISTS" --console=plain -q 2>/dev/null

Raw class: org.apache.cassandra.cql3.statements.UpdateStatement$ParsedUpdate (confirm via inspector)
Relevant fields: qualifiedName, a list of SET operations (column -> Term.Raw / operation),
  whereClause, attrs (TTL/timestamp), ifExists, conditions.

IMPLEMENTATION
1. New handler UpdateHandler implements CqlHandler<UpdateStatement.ParsedUpdate>.
2. Resolve keyspace/table (InvalidQueryException if missing).
3. Match rows using the WHERE logic from plan c. UPDATE in Cassandra requires the full primary key
   to be restricted by equality; enforce that (see FIDELITY).
4. For matched rows apply SET assignments (resolve marker/literal values, coerce to column DataType).
   If no row matches and it is a plain UPDATE (no IF), Cassandra performs an upsert -> create the row
   from the WHERE primary-key equalities plus the SET values. Mirror that.
5. IF EXISTS / IF <condition>: apply LWT semantics (only update when the row/condition holds); if PK
   plumbing is incomplete, implement the non-conditional path and leave a TODO.
6. Register UpdateHandler in the CqlHandlerRegistry list in SeaStarCqlRequestHandler.

FIDELITY (verify against ContainerCqlSessionTest)
  - WHERE not restricting the full primary key by = / IN -> InvalidQueryException.
  - Updating a primary key column in SET -> InvalidQueryException.
  - Unknown keyspace/table/column -> InvalidQueryException.

TESTS (AbstractCqlSessionTest)
  Insert a row, UPDATE it, SELECT and assert new value. UPDATE a non-existent PK (upsert) then SELECT.
