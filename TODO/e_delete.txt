PLAN: Support DELETE ... WHERE (DeleteHandler)

DEPENDS ON: a_create_table_persistence, b_insert, c_select_where (reuses WHERE matching).

READ FIRST
- Skill: .claude/skills/add-cql-statement/SKILL.md
- VolatileTable.removeRowIf(Predicate<SeaStarRow>)  <-- already exists, use it.
- SelectHandler WHERE logic (plan c).

INSPECT
  ./gradlew :lib:inspectRaw -Pquery="DELETE FROM foo.users WHERE user_id = ?" --console=plain -q 2>/dev/null
  ./gradlew :lib:inspectRaw -Pquery="DELETE first_name FROM foo.users WHERE user_id = ?" --console=plain -q 2>/dev/null

Raw class: org.apache.cassandra.cql3.statements.DeleteStatement$Parsed (confirm via inspector)
Relevant fields: qualifiedName, whereClause, a list of deleted columns/selections (empty = whole row),
  ifExists, conditions, attrs.

IMPLEMENTATION
1. New handler DeleteHandler implements CqlHandler for the DELETE parsed type.
2. Resolve keyspace/table (InvalidQueryException if missing).
3. Row-level delete (no column list): table.removeRowIf(row matches WHERE via plan c logic).
4. Column-level delete (DELETE col FROM ...): for matched rows set the named columns to null
   rather than removing the row.
5. Register DeleteHandler in the CqlHandlerRegistry list in SeaStarCqlRequestHandler.

FIDELITY (verify against ContainerCqlSessionTest)
  - WHERE must restrict the primary key by = / IN -> else InvalidQueryException.
  - Deleting/naming a primary key column in the column list -> InvalidQueryException.
  - Unknown keyspace/table/column -> InvalidQueryException.

TESTS (AbstractCqlSessionTest)
  Insert rows, DELETE one by PK, SELECT and assert it is gone and others remain.
  Column delete: assert the column is null but the row still exists.

VERIFY
  ./gradlew :lib:test --tests 'com.tagadvance.seastar.ContainerCqlSessionTest'
  ./gradlew :lib:test --tests 'com.tagadvance.seastar.SeaStarCqlSessionTest'
