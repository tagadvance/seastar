PLAN: Finish CREATE TABLE column persistence (CreateTableHandler)

WHY FIRST: This is foundational. SELECT, INSERT, UPDATE, DELETE all need real columns,
partition keys, and clustering columns to exist on the table. Today CreateTableHandler
parses columns but persists nothing (note the stray System.gc() and commented-out addColumn).

Raw class: org.apache.cassandra.cql3.statements.schema.CreateTableStatement$Raw
Relevant fields (read package-private ones via Reflections.getDeclaredField):
  Map    rawColumns          ColumnIdentifier -> CreateTableStatement$ColumnProperties$Raw
                             (each has rawType -> CQL3Type.Raw, and rawMask which must be null)
  List   partitionKeyColumns List<ColumnIdentifier>
  List   clusteringColumns   List<ColumnIdentifier>
  LinkedHashMap clusteringOrder  ColumnIdentifier -> ordering (ASC/DESC)
  Set    staticColumns       Set<ColumnIdentifier>
  boolean ifNotExists, useCompactStorage
Note: rawColumns preserves declared order (LinkedHashMap); persist columns in that order.

IMPLEMENTATION
1. In CreateTableHandler.processCql, after ksx.newSeaStarTable(table), for each rawColumns entry:
   - reject rawMask != null with UnsupportedOperationException (already done).
   - convert the column's rawType via SeaStarRawType.from(raw).toDataType()  (NOT .type();
     .type() returns a CQL3Type, toDataType() returns the driver DataType the model needs).
   - determine isStatic = staticColumns.contains(columnIdentifier).
   - table.addColumn(CqlIdentifier.fromInternal(colId.toString()), dataType, isStatic).
2. Persist key roles. VolatileTable currently has getPartitionKey()->List.of() and
   getClusteringColumns()->Map.of() as FIXMEs. Extend VolatileTable to record:
   - partition key column identifiers (ordered),
   - clustering column identifiers + ClusteringOrder (default ASC; DESC if clusteringOrder says so).
   Wire getPartitionKey() and getClusteringColumns() to return the real ColumnMetadata,
   looking columns up by name. Add setter/marker methods on SeaStarTable as needed
   (e.g. markPartitionKey(CqlIdentifier), markClustering(CqlIdentifier, ClusteringOrder))
   OR pass them into newSeaStarTable. Keep it thread-safe (write under the table lock).
3. Map CQL clustering order to com.datastax.oss.driver.api.core.metadata.schema.ClusteringOrder.
